# PULSE - Mode Autonome

## Comportement par Défaut

Lorsque j'active en mode autonome (`--headless`), je :
1. Formate automatiquement le code existant
2. Lance les tests automatisés
3. Documente les décisions techniques récentes
4. Met à jour la mémoire du projet

## Tâches Nommées

| Nom de Tâche | Action |
|-------------|--------|
| format-code | Formate tout le code du projet |
| run-tests | Lance tous les tests automatisés |
| update-docs | Met à jour la documentation technique |

## Fréquence

- **Formatage du code** : À chaque activation autonome
- **Tests automatisés** : À chaque activation autonome
- **Mise à jour documentation** : Une fois par jour

## Heures Calmes

- **Nuit** : 22h00 - 06h00 (pas d'activation autonome)
- **Week-end** : Activité réduite

## Priorités

1. **Intégrité du code** - S'assurer que les tests passent
2. **Qualité** - Formater et documenter
3. **Mémoire** - Maintenir à jour l'historique
